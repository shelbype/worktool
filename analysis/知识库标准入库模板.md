# 知识库标准入库模板

## 使用目标

这套模板用于后续人工编写知识库内容，重点服务客服问答和 RAG 检索。每一行只写一个明确问题，答案必须能直接用于客服回复，不能把宣传语、页面标题、导航标题、图片说明当作回答正文。

## 文件

- `知识库标准入库模板.csv`: 带样例行，适合参考写法。
- `知识库标准入库空白模板.csv`: 只有表头，适合复制后正式填写。

正式导入前，请删除 `知识ID` 以 `样例-` 开头的示例行。

## 推荐字段说明

| 字段 | 是否必填 | 写法要求 |
| --- | --- | --- |
| 知识ID | 必填 | 建议格式为 `kb-业务-四位序号`，例如 `kb-homework-0001`。 |
| 入库状态 | 必填 | 可填 `待复核`、`可入库`、`暂不入库`、`已入库`。 |
| 适用端口 | 必填 | 用中文分号分隔，例如 `老师；教务`。 |
| 角色库 | 必填 | 对应问答库，例如 `老师库；学生库`。 |
| 一级分类 | 必填 | 建议使用稳定分类，例如 `作业/布置/批改/反馈`、`浏览器/网络/缓存`。 |
| 二级分类 | 建议填 | 具体场景，例如 `学生看不到作业`、`音频加载异常`。 |
| 问题标题 | 必填 | 写成用户会问的问题，不要只写功能名。 |
| 用户问法示例 | 必填 | 写 3 到 8 个相近问法，用中文分号分隔。 |
| 意图关键词 | 必填 | 写召回关键词，包括同义词、报错词、功能名。 |
| RAG检索摘要 | 必填 | 1 句话概括这条知识解决什么问题。 |
| 标准回答 | 必填 | 可直接发给客户的自然语言回答，先给结论，再给操作。 |
| 操作步骤 | 建议填 | 如果有后台路径或排查流程，按 `1. 2. 3.` 写清楚。 |
| 前置条件 | 建议填 | 说明这条答案成立的条件。 |
| 排查项 | 建议填 | 用于客服快速判断问题原因。 |
| 注意事项 | 建议填 | 写限制、容易误解的点、不要误导客户的点。 |
| 不适用场景 | 建议填 | 明确哪些情况不能套用本答案。 |
| 转人工条件 | 必填 | 写清楚什么时候不能自动回答，需要人工介入。 |
| 关联知识ID | 可选 | 填相关知识 ID，多个用中文分号分隔。 |
| 来源类型 | 必填 | 例如 `帮助中心`、`客户反馈沉淀`、`产品规则`、`人工整理`。 |
| 来源ID/链接 | 建议填 | 方便回查来源。 |
| 图片引用 | 可选 | 只放图片链接或 JSON 元数据，不放入标准回答。 |
| 维护人 | 必填 | 负责后续维护的人。 |
| 复核状态 | 必填 | 可填 `待复核`、`已复核`、`需改写`。 |
| 更新时间 | 必填 | 建议格式 `YYYY-MM-DD`。 |
| 质量标签 | 建议填 | 例如 `标准回答；可操作；高频问题`。 |
| 内部备注（不入检索正文） | 可选 | 只给内部看，不参与 RAG 检索和客服回答。 |

## 标准回答写法

推荐结构：

1. 先回答结论。
2. 再给排查或操作步骤。
3. 最后说明异常时需要提供什么信息或转人工。

示例：

```text
老师，学生看不到作业时，建议先检查学生是否在对应班级内、作业布置对象是否包含该学生，以及学生账号是否具备做题权限。确认无误后，让学生刷新页面或重新登录再查看。若仍看不到，请提供学生账号、作业名称和截图，我们再进一步排查。
```

## 不要入库的内容

以下内容不要写进 `标准回答`：

- 宣传语，例如“让教学更高效”“开启智能教学新时代”。
- 页面标题或目录标题，例如“软件安装”“核心功能介绍”。
- 只有功能名、没有解决方案的短句。
- 图片占位，例如 `[图片: xxx]`。
- 图片说明，例如 `图片说明：点击这里进入后台`。
- 内部处理痕迹，例如“技术已处理”“已提需求”，除非这是用户可感知的正式规则。
- 没有边界的泛化回答，例如“建议刷新一下看看”。

## RAG 质量规则

人工编写时建议遵守这些规则：

1. 一条知识只解决一个问题，不要把多个无关功能写在一行。
2. 问题标题要像用户真实提问，不要写成后台菜单名。
3. 用户问法示例要覆盖口语化表达、错别字高频表达和功能名表达。
4. 标准回答中不要依赖“上图”“这里”“这个按钮”等无上下文指代。
5. 能写路径就写路径，例如 `教务端 -> 学员管理 -> 编辑 -> 续期`。
6. 有权限限制、角色限制、设备限制时必须写清楚。
7. 有转人工条件时必须明确，避免模型强答。
8. 图片只作为元数据保留，不进入可检索正文。

## 入库前检查清单

- `标准回答` 是否能直接发给客户。
- 是否去掉了宣传语、标题噪声、图片说明。
- 是否写清楚适用端口和角色库。
- 是否有 3 个以上用户问法示例。
- 是否有关键词便于召回。
- 是否写清楚不适用场景和转人工条件。
- 是否和已有知识重复或冲突。
- 是否有人复核过答案准确性。
