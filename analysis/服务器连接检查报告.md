- `d:\worktool\.ssh-deploy\worktool_deploy_ed25519`

- 注释：`worktool-deploy`
- 指纹：`SHA256:ILC5Kw8oPAEfO6qBob1yox36Jdk+DpUtuPG3w526dnE`
- 公钥：

```text
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIHuyOQbTsNI9uinzDgoJbF2Af42ljKUq9JjI+C4IDGbE worktool-deploy
```

已尝试的用户：

- `root`
- `worktool`
- `ubuntu`
- `ecs-user`
- `admin`

结果均为：

```text
Permission denied (publickey,gssapi-keyex,gssapi-with-mic,password).
```

SSH 详细日志显示，客户端已经向 `root@139.196.6.90` 提交了 `worktool-deploy` 公钥，但服务端没有接受该 key。因此目前只能通过线上 HTTP API 检查服务状态，不能直接登录服务器执行数据库清理或入库。

建议在阿里云控制台或服务器远程终端中确认这把公钥是否已经绑定到实例，或是否已写入：

```text
/root/.ssh/authorized_keys
```

如果使用阿里云控制台的“远程连接”进入服务器，可以执行：

```bash
mkdir -p ~/.ssh
chmod 700 ~/.ssh
grep -q "worktool-deploy" ~/.ssh/authorized_keys 2>/dev/null || echo 'ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIHuyOQbTsNI9uinzDgoJbF2Af42ljKUq9JjI+C4IDGbE worktool-deploy' >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
chown -R root:root ~/.ssh
grep -n "worktool-deploy" ~/.ssh/authorized_keys
```

配置完成后，本机应可用下面命令登录：

```powershell
ssh -i .ssh-deploy\worktool_deploy_ed25519 root@139.196.6.90
```

## 不能直接用 HTTP 完成重新入库的原因

线上存在 `/knowledge/documents` 接口，但它的语义是单篇文档摄入，入参只有：

- `source_id`
- `title`
- `body_html`
- `category`
- `product_module`
- `product_version`
- `source_url`
- `image_urls`

没有看到以下能力：

- 清空旧知识库
- 清空各角色库
- 全量替换导入
- 导入前备份
- 删除旧噪声 chunk

因此不建议用这个接口直接追加新数据。追加会让旧脏数据继续参与检索，无法解决“续课问题命中麦克风内容”这类误召回。
