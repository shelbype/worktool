## 建议用途

### 1. 作为“待审核知识池”的主要补充来源

优先导入：

- `问题类型 = 操作疑问`
- `问题类型 = 使用问题`
- `处理进度 = 已处理`
- `问题回答` 不为空
- 不包含退费、合同、发票、报价、赔偿等高风险词

这部分大约有 1118 条可作为第一批候选，但仍建议人工抽检后再进入正式库。

### 2. 作为评测集扩充来源

客户原话很适合扩充 `eval/questions.json`，尤其是：

- 登录不上账号
- 学生作业看不到
- 音频加载失败
- 学生换手机号登录不了
- 销售看不到入学模考结果
- 老师上完课没有待反馈
- 学杂费/课程/订单字段为空
- 公池权限看不到数据

这些问题比帮助文档更接近真实群聊表达，能提高评测对线上效果的约束力。

### 3. 作为五类路由校准数据

`端口` 字段可直接映射：

```text
学生/家长 -> student
老师/助教 -> teacher
校长 -> principal
教务 -> academic
销售/市场/财务 -> sales
```

可以用它反向评估 `AudienceRouter`，发现“学生怎么查看作业”这类多角色问题应该查多个库。

### 4. 作为固定转人工/工单规则来源

以下类型不要自动答，优先转人工或生成工单：

- `bug`
- `题目问题`
- `机构题目问题`
- `数据恢复需求`
- `录入需求`
- `入学模考配置`
- `需求`
- `优化`
- `查询需求`

这类问题约 3117 条，价值不在自动回复，而在识别“机器人不能答”的边界。

## 不建议直接入正式知识库的原因

很多记录是客户现场问题，不是通用知识：

- 某个学生、某个账号、某个班级的具体异常
- 题目答案错误、录题需求、模考配置需求
- 数据恢复、删除恢复、后台协助
- bug 或未复现问题
- 合同、退费、收费、发票等高风险问题

如果直接进入正式库，容易造成机器人误答或承诺过度。

## 推荐实施顺序

1. 新增反馈记录导入命令：

```bash
python -m ragbot.cli import-feedback-candidates --source db/init/客户反馈记录.xlsx
```

2. 导入后默认进入审核池，不参与正式召回。

3. 审核页面或接口里显示：

```text
客户问题
问题描述
问题回答
端口
问题类型
建议 audience
附件
质量分桶
```

4. 审核通过时改写成标准知识：

```text
标题：学生做题音频加载失败怎么办
适用端口：学生/老师
问题说法：音频加载中、听力没有声音、听力播放失败
处理步骤：
1. 优先确认浏览器，建议使用 Chrome。
2. 刷新页面或清理缓存后重新进入。
3. 若音频超过 15 秒未加载，可按页面提示下载音频本地播放。
4. 若仍失败，提供学生账号、测试链接、截图转人工排查。
```

5. 审核通过后写入对应五类物理表。
